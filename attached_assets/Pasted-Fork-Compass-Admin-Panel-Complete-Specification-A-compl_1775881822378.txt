1. Overview & Tech Stack
The admin panel is a single-page React application, hosted at the path /admin.

Concern	Technology
Framework	React 18 + TypeScript
Build tool	Vite
Routing	Wouter
Data fetching	TanStack Query (caching + revalidation)
Styling	Tailwind CSS
UI primitives	Custom component library (src/components/ui/) modelled on shadcn/ui
Icons	Lucide React
Forms	Local useState — no form library
Package	@workspace/admin
Base URL: All API calls use /api as the base, proxied to the API server at :8080.

2. Routing Structure
/admin/             → DashboardPage     (inside AppShell)
/admin/login        → LoginPage         (standalone, no shell)
/admin/recipes      → RecipesPage       (inside AppShell)
/admin/recipes/:id  → RecipeEditPage    (inside AppShell)
/admin/users        → UsersPage         (inside AppShell)
/admin/users/:id    → UserDetailPage    (inside AppShell)
/admin/featured     → FeaturedPage      (inside AppShell)
/admin/settings     → SettingsPage      (inside AppShell)

⚠️ Current gap: There is no authentication guard. Any user can navigate to /admin without logging in. LoginPage exists and submits to POST /api/admin/login but App.tsx does not wrap any routes in a ProtectedRoute component.

Tier Completion Logic
// TierBadge counts tiers across all steps
function getTierCount(steps: RecipeStep[]): { filled: number; total: 3 } {
  const hasFirstSteps = steps.every(s => s.instructionFirstSteps?.trim());
  const hasStandard = steps.every(s => s.instruction?.trim());
  const hasChefsTable = steps.every(s => s.instructionChefsTable?.trim());
  return {
    filled: [hasFirstSteps, hasStandard, hasChefsTable].filter(Boolean).length,
    total: 3
  };
}

Color coding: 3/3 = green, 2/3 = amber, 1/3 = red, 0/3 = grey

Row Actions (dropdown ⋮)
Action	API call	Behaviour
Edit	—	Navigates to /admin/recipes/:id
Feature / Unfeature	POST /api/admin/recipes/:id/feature	Toggles featured boolean
Hide / Unhide	PATCH /api/admin/recipes/:id/status	Sets status to "hidden" or "live"
Duplicate	POST /api/admin/recipes/:id/duplicate	Creates copy with " (Copy)" appended to title
Delete	DELETE /api/admin/recipes/:id	Confirm dialog → deletes
Bulk Actions
Appear in a bar above the table when 1+ rows are selected.

Button	API call
Hide Selected	PATCH /api/admin/recipes/bulk { ids, status: "hidden" }
Unhide Selected	PATCH /api/admin/recipes/bulk { ids, status: "live" }
Delete Selected	DELETE /api/admin/recipes/bulk { ids } — confirm dialog first

Page Actions
Button	API call	Behaviour

Delete	DELETE /api/admin/recipes/:id	Confirm dialog → deletes → navigate back to list
Back	—	Navigates to /admin/recipes
Dirty state tracking: A dirty boolean is set true on any field change. Navigating away with unsaved changes shows a browser confirm() dialog.

14. Data Interfaces
Full Recipe (from API)
interface AdminRecipe {
  id: string;                     // kebab-case, e.g. "classic-garlic-bread"
  title: string;                  // Display name
  description: string;
  countryId: string;
  countryName: string;
  region: string;                 // Sub-region
  category: string;               // "Main Course", "Side Dish", etc.
  difficulty: "Easy" | "Intermediate" | "Advanced";
  prepTime: string;               // "20 min"
  cookTime: string;               // "0 min"
  servings: number;
  culturalNote: string;
  image: string;                  // URL (relative or absolute)
  status: "live" | "hidden" | "draft";
  featured: boolean;
  featuredOrder: number;          // 1-5, lower = earlier in carousel
  cookCount: number;              // How many users have cooked this
  tips: string[];
  ingredients: Ingredient[];
  steps: RecipeStep[];
  createdAt: string;              // ISO timestamp
}
interface Ingredient {
  id: string;
  name: string;
  amount: string;                 // Free-text, e.g. "115 g softened"
}
interface RecipeStep {
  id: string;                     // "s1", "s2", etc.
  title: string;                  // Short verb, e.g. "prep"
  instruction: string;            // Standard / Home Cook tier
  instructionFirstSteps: string;  // Beginner tier
  instructionChefsTable: string;  // Advanced / Chef tier
  materials: string[];            // Ingredient names used in this step
}

User (from API)
interface AdminUser {
  id: string;
  email: string;
  name: string;
  cookingLevel: "beginner" | "intermediate" | "advanced";
  subscriptionPlan: "free" | "pro" | "premium";
  recipesCooked: number;
  cuisinesExplored: number;
  measurementSystem: "imperial" | "metric";
  temperatureUnit: "fahrenheit" | "celsius";
  groceryPartner: "instacart" | "kroger" | "walmart";
  joinedAt: string;
  lastActiveAt: string;
  history: CookingHistoryEntry[];
  feedback: FeedbackEntry[];
}
interface CookingHistoryEntry {
  recipeId: string;
  recipeTitle: string;
  completedAt: string;
  rating: number;               // 1-5
  cookTimeMinutes: number;
}
interface FeedbackEntry {
  recipeId: string;
  recipeTitle: string;
  comment: string;
  rating: number;
  createdAt: string;
}
