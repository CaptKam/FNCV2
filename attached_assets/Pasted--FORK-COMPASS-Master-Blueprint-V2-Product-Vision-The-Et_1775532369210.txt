# 1. Product Vision

Fork & Compass is a premium culinary travel app built on a single premise: you can travel the world from your kitchen. Pick a country. Plan a dinner. Shop the ingredients. Cook the meal. Feel like you traveled.

This is not a recipe database. It is a cultural experience engine that happens to involve cooking. Every interaction should feel like opening a luxury food magazine and stepping inside.

Born during COVID, when friends would pick a country and build an entire evening around its cuisine, drinks, and music. That dinner party energy — the planning, the anticipation, the shared experience — is the emotional core.

## Who this is for

Serious home cooks who see cooking as exploration, not obligation. The user who spends Sunday afternoon planning the week's meals, who browses recipes the way others browse travel blogs, and who treats a dinner party as an event worth orchestrating.

## Competitive position

Fork & Compass owns culturally authentic world cuisine for serious home cooks. The moat is cultural depth combined with the travel premise plus backward-planning dinner timeline coordination.

- NYT Cooking: American/European bias, no cultural travel framing
- SideChef: pivoted to grocery commerce, lost editorial soul
- HelloFresh: validated international cooking at home, but commoditized
- Mela / Paprika: utility-first, no cultural depth or social features

# 2. Design Philosophy

## 2.1 — The Three Governing Frameworks

Every pixel is governed by Steve Jobs' functional aesthetics, Apple's Human Interface Guidelines (including iOS 26 Liquid Glass), and the editorial sensibility of luxury print magazines.

**Design is how it works.** Beautiful food photography means nothing if the user cannot figure out how to plan their week. A glassmorphic tab bar is worthless if it obscures the content it frames. Every aesthetic choice must serve a functional purpose. If a visual element does not help the user accomplish their goal, it is removed.

## 2.2 — The Subtraction Audit

Every screen, feature, and interaction passes through three tests:

| Test | Question | If it fails |
|------|----------|-------------|
| Mission-critical | What is the ONE thing the user needs here? | Remove or deeply subordinate |
| Grandma test | Could a non-technical user complete this without explanation? | Redesign until no instruction is needed |
| Day-better test | Does this genuinely make someone's day better? | Cut it, regardless of engineering investment |

## 2.3 — The Four HIG Pillars

| Principle | Fork & Compass application |
|-----------|---------------------------|
| Clarity | Recipe cards show prep time, difficulty, and country at a glance. No decoding required. |
| Deference | Food photography is the hero. Chrome is minimal. Glass effects frame content, never compete. |
| Depth | Modal sheets slide up for cook mode. Plan layers weekly over daily. Spatial relationships always clear. |
| Consistency | Tab bar always bottom. Back always top-left. Swipe gestures identical everywhere. |

## 2.4 — The iOS 26 Triad

| Principle | Implementation |
|-----------|---------------|
| Hierarchy | Cook mode suppresses all navigation chrome. Plan surfaces next upcoming meal. Discover hides metadata until scroll past hero. |
| Harmony | Tab bar glass matches device edge curves. Content insets respect Dynamic Island and home indicator. |
| Consistency | Same interaction model across iPhone SE and 16 Pro Max. No features hidden on smaller screens. |

## 2.5 — Non-Negotiable Principles

1. Design is how it works. If beautiful but confusing, it has failed.
2. Subtract before you add. Every element must earn its place.
3. Content is the hero. Food photography and cultural stories, not chrome and UI widgets.
4. Finish the back of the cabinet. The codebase, API, admin — invisible quality everywhere.
5. Follow the HIG. Tab bars at bottom. Back at top-left. Standard gestures. No exceptions.
6. Glass is for navigation, never for content. Liquid Glass frames the experience. Food stays crisp.
7. Speak like a human. Warm, active, culturally literate. Never robotic. Never blame the user.
8. Accessibility is architecture. VoiceOver, Dynamic Type, 48pt touch targets, reduced motion, color independence.
9. Say no to protect the yes. Features are deferred, not deleted. Deferred means not built today.

# 3. The Ethereal Archivist Design System

The Creative North Star is "The Ethereal Archivist" — a sophisticated intersection of heritage warmth and spatial computing futurism. Terracotta and cream ground the human element. Liquid Glass brings 2026 depth. The result is an interface that feels like a physical object caught in digital light.

## 3.1 — Foundational Rules

**The No-Line Rule:** 1px solid borders for sectioning are strictly prohibited. Structure is defined through background color shifts and whitespace. If you feel a border is needed, you haven't used the surface tiers correctly.

**The Layering Principle:** Treat the UI as stacked sheets of fine paper and glass. Depth comes from tonal layering, not drop shadows. Shadows are a last resort.

**The Editorial Pattern:** Every section header uses "Caption over Title" — a tiny uppercase Inter label above a large Noto Serif headline. This is the fingerprint of the Ethereal Archivist.

**Glass is for navigation only:** Liquid Glass effects apply to the tab bar, toolbars, floating CTAs, and sheet headers. Never to content itself. Food photography and recipe text remain crisp.

Shadows are a last resort. Reach for tonal layering first.

| Level | Offset | Opacity | Radius | Usage |

## 3.7 — Liquid Glass Specifications

The hallmark of the Ethereal Archivist. Applied to navigation layer only.

### Light variant
- Background: rgba(255, 255, 255, 0.7)
- Backdrop blur: 32px (BlurView intensity 70)
- Specular highlight: 1px top border, rgba(255,255,255,0.4)
- TOP EDGE ONLY. All four sides flattens the effect.

### Dark variant
- Background: rgba(29, 27, 24, 0.85)
- Backdrop blur: 40px
- Specular highlight: 1px top border, rgba(255,255,255,0.15)

## 3.8 — Motion and Haptics

All animations respect useReducedMotion(). When enabled, every transition becomes instant crossfade. No exceptions.

| Interaction | Duration | Easing | Haptic |
|-------------|----------|--------|--------|
| Button press | 150ms | ease-in (scale 0.97) | Light |
| Primary CTA tap | 200ms | spring (damping: 15) | Medium |
| Navigate forward | 320ms | ease-out (slide-right) | None |
| Navigate back | 320ms | ease-in (slide-left) | None |
| Bottom sheet present | 400ms | cubic-bezier ease-out | Light |
| Recipe bookmark toggle | 250ms | spring (damping: 12) | Light |
| Cook mode step swipe | 280ms | spring (damping: 18, stiffness: 120) | Medium |
| Timer completion | Instant | N/A | Success notification |
| Skeleton shimmer | 1.5s loop | ease-in-out | None |

Spring-based physics via react-native-reanimated. No linear easing ever on visible UI.

# 4. App Architecture

## 4.1 — Five-Tab Structure

The tab bar is a FLOATING PILL — not a standard iOS bottom bar. Centered, pill-shaped (radius.full), 90% width, positioned 32px from bottom. Liquid Glass Regular variant with ambient shadow.

| Tab | Icon | Purpose |
|-----|------|---------|
| Discover | explore (filled when active) | Browse countries and featured recipes |
| Search | search | Find specific recipes, filter by mood/diet |
| Plan | calendar_today | Weekly/daily meal planning |
| Grocery | shopping_cart | Shopping list and delivery prep |
| Cook | restaurant | Guided cooking with timeline coordination |

Profile/Settings accessed via avatar icon in header, not tab bar.

## 4.2 — Navigation Rules

- No hamburger menus. Tab bar always visible (exception: full-screen cook mode).
- No deep modal stacking. Modals for focused tasks only.
- No hidden gesture-only actions. Every swipe has a visible button equivalent.
- No tab icons that change meaning by context.
- Every StatusBar responds to current color scheme.

## 4.3 — Screen Flow

```
Welcome Gate → Onboarding (Country Picker) → Tab Shell

From Tab Shell:
  Discover → Country Detail → Recipe Detail → Cook Mode
  Search → Recipe Detail → Cook Mode
  Plan → Recipe Detail / Dinner Setup → Cooking Schedule → Cook Mode
  Grocery → In-Store Mode
  Cook → Active Session / Cook Mode
```

# 5. Feature Specifications

## 5.1 — Discover Tab

Food photography first. Minimal text. Maximum visual impact. The user's eye captured by gorgeous imagery before reading a single word.

**Hero carousel:** Full-bleed 520px images, one per country, horizontal paginated scroll. Manual swipe only (no auto-rotate). Gradient overlay. Country flag in liquid-glass pill badge. Country name in displayMedium (white). "Explore [Country]" terracotta CTA pill. Vertical pagination dots on right side. Country destination rings (64px) at bottom of hero with liquid glass treatment.

**Tonight's Plan strip:** Liquid glass card below hero. Food thumbnail + "TONIGHT'S PLAN" label + recipe name + "Cook Tonight" link. Bridges Discover to Cook.

**Curated Regions grid:** 2-column destination cards (320px tall, rounded-xl). Full-bleed images with gradient overlay. Country label + region name. Image zoom on hover (scale 1.1, 700ms).

**Trending Bites grid:** 2-column recipe cards (surfaceContainer background, rounded-[28px]). Food image + liquid glass heart button + recipe name + time badge + "Tonight" / "Week" quick-add buttons.

**No cultural quotes. No editorial paragraphs.** Cultural depth lives on the Country Detail screen, not here.

## 5.2 — Search Tab

Functional, clean, fast.

**Search bar:** Full width, 52pt height, surfaceContainerLow fill, no border. Placeholder: "Ingredients, dishes, or moods..."

**Filter chips:** Horizontal scroll. Active chip: primary fill, white text. Inactive: surfaceContainerHigh fill. Mood-based: All Moods, Adventurous, Comfort, Exotic, Quick.

**Results:** 2-column food-first grid. Large images, recipe name in headline, metadata in labelSmall, "ADD +" quick-add action, heart button. Results span both recipes AND countries.

**Recent searches:** Shown when search field is empty.

## 5.3 — Plan Tab (The Core Innovation)

The Sunday planning ritual. This is where browsing becomes committing.

**Calendar strip (top):** 7-day compact overview. Day letter + day number. Today highlighted with secondaryContainer background. Terracotta dot if meal planned, outlineVariant dot if empty. Tapping a day scrolls the timeline below.

**Timeline (scrollable body):** Vertical transit line (2px, primary at 20%) with terracotta node dots per day. Each day shows date + optional dinner party badge.

**Filled day card:** Liquid glass card. Food image (160px) + meal course badge (DINNER/APPETIZER/DESSERT) + recipe name + cook time + calories.

**Multi-course slots:** Daily view shows Appetizer → Main → Dessert as separate tappable slots. Empty slots show dashed-border "blank ticket" with course label + add button.

**Auto-Generate:** Floating action button (56x56, primary fill, sparkle icon, radius.full). User selects which days to cook via calendar strip dots, taps Auto-Generate, system fills ONLY those days. Respects course count from Settings.

**Grocery alert banner:** Shows missing ingredient count with "Review" link to Grocery tab.

**Ready to Cook bar:** Floating liquid glass pill above tab bar. Play icon + "Ready to Cook" + day label.

**Dinner party indicator:** Per-day icon. Muted (onSurfaceVariant) when no party. Terracotta (primary) when party exists.

## 5.4 — Grocery Tab

Auto-populated from planned recipes. The user never manually enters categories.

**Mode toggle:** Online / In-Store pill toggle at top.

**Recipe source cards:** Horizontal scroll showing which recipes generated the list items. Each card: food image + recipe name + plan badge ("DINNER PLAN").

**Category-grouped list:** Produce, Protein, Dairy, Pantry, Spice. Emoji + category name as headers. No divider lines — whitespace between groups.

**Item rows:** surfaceContainerLow background, radius.md. Circle checkbox (outlineVariant when unchecked, success fill + checkmark when checked). Ingredient name + recipe source ("Used in 2 Recipes") + amount + aisle number. Checked items: strikethrough + "IN PANTRY" label.

**Ingredient deduplication:** 2 cloves (French recipe) + 3 cloves (Italian) = 5 cloves garlic on the list.

**Tab badge:** Unchecked item count shown on Grocery tab icon.

**Order footer:** Sticky bottom bar. Terracotta gradient. Item count + subtotal + "ORDER FROM INSTACART" CTA.

## 5.5 — Cook Tab

The flagship dashboard.

**Reputation header:** Avatar + "KITCHEN REPUTATION" label + skill level name (Home Cook) + XP progress bar + level count.

**Active cooking card:** Large food image + "IN THE KITCHEN" label + recipe name + step progress ("Step 3 of 8") + "Resume Session" CTA.

**Technique Library:** Horizontal scroll of video-style technique cards with dark overlay + duration badge.

## 5.6 — Cook Mode (Full-Screen Modal)

The technical crown jewel. The most intuitive guided cooking experience in a mobile app.

**Environment:** Full-screen modal. Dark background (inverseSurface #32302C for kitchen contrast). expo-keep-awake active. Screen never dims.

**Top bar:** Close button (X, 48pt) + recipe name in titleMedium + step counter in caption.

**Progress bar:** 3pt height strip. Primary fill for completed, outlineVariant for remaining.

**Step content:** Large instruction text in body (20px for kitchen legibility, white). Per-step ingredient pills (secondaryContainer fill, radius.sm). Timer display when step has duration.

**Bottom toolbar:** Liquid Glass Dark variant. Previous/Next buttons (52pt height). Swipe left/right with haptic feedback for step navigation.

**Adaptive cooking language:** Three tiers of instruction voice, same ingredients.

| Tier | Icon | Example |
|------|------|---------|
| First Steps | 🌱 | Carefully cut the onion into small pieces about the size of your fingernail. |
| Home Cook | 🍳 | Dice the onion finely. |
| Chef's Table | 👨‍🍳 | Brunoise the onion. |

**Step timers:** Per-step with Reanimated transitions. Success haptic on completion.

**Servings scaler:** Adjust servings, all quantities recalculate instantly.

**CookingPill:** Floating indicator on tab bar when cook session is active. Shows recipe name + timer. Tapping returns to cook-mode.

## 5.7 — Country Detail Screen

Opens from tapping a destination card on Discover.

Hero image (360px) + gradient overlay + country name + flag badge. Back button in liquid glass circle.

Below hero: "CULINARY HERITAGE" label + "[Country] Cuisine" headline + 2-3 short paragraphs about the cuisine tradition. Then recipe list from that country: vertical rows with 80x80 image + recipe name + cook time + difficulty.

Cultural depth lives here — not on the Discover tab.

## 5.8 — Recipe Detail Screen

Opens from tapping any recipe card.

Hero food image (380px) + gradient overlay + recipe name + country flag. Stats row: prep time, cook time, difficulty, servings. Servings adjuster.

Ingredients grouped by category with labelLarge headers. Instruction steps numbered with primary circles. Timer icons where applicable. Cultural note card (surfaceContainerLow, left border accent in primary).

"Start Cooking" fixed bottom CTA (full width, primary pill, liquid glass backdrop).

## 7.2 — Key Libraries

expo-image, expo-haptics, expo-blur, expo-linear-gradient, expo-keep-awake, react-native-reanimated, expo-router, @tanstack/react-query.

## 7.3 — State Management

| Context | Purpose |
|---------|---------|
| AppContext | The brain. Cook sessions, dinner plans, grocery state, itinerary, saved recipes, dietary flags, cooking level, appearance mode. Persists to MMKV/AsyncStorage. |
| AuthContext | Supabase auth. Injects Bearer token into API client. |
| SubscriptionContext | RevenueCat gate. Currently isPremium = true. |

Storage fallback chain: MMKV (fast, synchronous, production) → AsyncStorage (async, Expo Go dev).

## 7.4 — Data

97 hand-curated recipes across 8 countries (Italy, France, Japan, Mexico, Thailand, Morocco, India, Spain). Sourced from recipe-api.com (paid plan), cached permanently. Embedded as TypeScript constants (520KB). Live API data merged with local fallback via useCountries/useRecipe hooks.

## 7.5 — Contract-First API

Endpoints defined once in lib/api-spec/openapi.yaml. Orval codegen produces both api-client-react (TanStack Query hooks) and api-zod (validation schemas). All frontends and backend consume the same types.

## 7.6 — Database Schema

| Table | Purpose |
|-------|---------|
| countries | Regional destinations |
| recipes | Core content (JSONB for ingredients and steps) |
| recipe_details | Extended data: nutrition, equipment |
| app_users | Profiles, cooking level, dietary preferences |
| dinner_parties | Party metadata, date, menu, host |
| dinner_guests | Guest list, RSVP status, dietary restrictions |

Currently empty — app runs on embedded constants. Define with Drizzle when connecting real data.
